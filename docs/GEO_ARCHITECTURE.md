# Архитектура Географии и Регионального Масштабирования (ЮФО)

## Обзор
Приложение «Услуги Рядом» использует **гибридную модель управления географией**. Вместо жестко заданного списка городов, система динамически расширяется на основе выбора пользователей, обеспечивая 100% покрытие Южного федерального округа (ЮФО).

## Текущая зона покрытия
Система настроена на работу исключительно в **Южном федеральном округе (ЮФО)**. Список поддерживаемых регионов:
- Ростовская область
- Краснодарский край
- Республика Адыгея
- Республика Калмыкия
- Республика Крым
- Город федерального значения Севастополь
- Астраханская область
- Волгоградская область
- Донецкая Народная Республика (ДНР)
- Луганская Народная Республика (ЛНР)
- Херсонская область
- Запорожская область

## Ключевые компоненты

### 1. API подсказок (DaData Proxy)
**Файл:** `apps/web/src/app/api/suggest/address/route.ts`
- Все запросы к DaData проходят через серверный прокси.
- В запрос принудительно вшивается фильтр по 8 регионам ЮФО.
- Это гарантирует, что пользователь не сможет выбрать адрес вне зоны обслуживания.

### 2. Динамическая регистрация пунктов (ensureCityAction)
**Файл:** `apps/web/src/features/order-creation/api/ensure-city-action.ts`
Вместо ошибки «Город не поддерживается», система выполняет следующие действия при выборе адреса:
1. Проверяет наличие населенного пункта в таблице `City` (по `fiasId` или имени/региону).
2. Если пункт не найден, он **автоматически создается**.
3. Для нового пункта генерируется уникальный `slug`.
4. **Универсальная привязка**: К новому пункту автоматически привязываются все активные категории из таблицы `Category`. Это гарантирует, что в любом хуторе сразу доступны все услуги.

### 3. Центральный конфигуратор (SSOT)
**Файл:** `apps/web/prisma/cities.config.ts`
- Содержит список «опорных» городов (столицы регионов и крупные центры).
- Используется скриптом `seed.ts` для первичного наполнения базы.
- Является эталоном для названий регионов и координат.

## Масштабирование на другие округа
Для расширения географии за пределы ЮФО необходимо:
1. Добавить новые регионы в массив `locations` в файле `api/suggest/address/route.ts`.
2. Добавить названия регионов в массив `sfoRegions` (его стоит переименовать в `allowedRegions`) в `ensure-city-action.ts`.
3. Запустить сид для добавления опорных городов новых регионов (опционально).

## Технические детали
- **Уникальность**: Города идентифицируются по `fiasId` (Dadata) для исключения дублей при совпадении названий в разных регионах.
- **Координаты**: Сохраняются в полях `lat`/`lng` для последующего отображения на картах.
- **Слаги**: Генерируются автоматически с добавлением случайного суффикса для обеспечения уникальности URL.
