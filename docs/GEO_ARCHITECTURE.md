# Архитектура географического охвата (Гео-модуль)

## Описание
Платформа использует гибридную модель данных для обеспечения масштабируемости на большие территории без необходимости ручного наполнения базы данных каждым хутором и поселком.

## Текущая зона покрытия (12 регионов)
- Ростовская область
- Краснодарский край
- Республика Адыгея
- Республика Калмыкия
- Республика Крым
- Город федерального значения Севастополь
- Астраханская область
- Волгоградская область
- Донецкая Народная Республика (ДНР)
- Луганская Народная Республика (ЛНР)
- Херсонская область
- Запорожская область

## Ключевые компоненты

### 1. Серверный фильтр (Proxy)
**Файл:** `apps/web/src/app/api/suggest/address/route.ts`
- Весь поиск адресов через DaData проксируется через этот роут.
- Применяется жесткий фильтр `locations` по регионам. Пользователь не сможет выбрать адрес вне разрешенной зоны.

### 2. Динамическая регистрация (ensureCityAction)
**Файл:** `apps/web/src/features/order-creation/api/ensure-city-action.ts`
- Когда пользователь выбирает населенный пункт из DaData, которого еще нет в нашей БД, этот экшен:
  1. Проверяет наличие населенного пункта в таблице `City` (по `fiasId` или имени/региону).
  2. Если пункт не найден, он **автоматически создается**.
  3. Для нового пункта генерируется уникальный `slug`.
  4. **Универсальная привязка**: К новому пункту автоматически привязываются все активные корневые категории (где `parentId: null`). Это гарантирует, что в любом хуторе сразу доступны все услуги.

### 3. Центральный конфигуратор (SSOT)
**Файл:** `apps/web/prisma/cities.config.ts`
- Содержит эталонный список крупных городов (административных центров) для первичного наполнения базы (Seed).

## Инструкция по модификации (Шпаргалка)

Если вам нужно расширить сервис на новые области или, наоборот, сузить географию, следуйте этой инструкции:

### 1. Как добавить новый регион
Нужно внести изменения в **два** файла:

1.  **`apps/web/src/app/api/suggest/address/route.ts`**:
    *   Найдите массив `locations`.
    *   Добавьте объект `{ region: "Название региона" }`. Это разрешит DaData показывать подсказки для этого региона.
    
2.  **`apps/web/src/features/order-creation/api/ensure-city-action.ts`**:
    *   Найдите массив `sfoRegions`.
    *   Добавьте название региона в **нижнем регистре** (например, `"московская"`). Это разрешит системе сохранять города этого региона в вашу базу данных.

### 2. Как изменить сообщение об ошибке
Если пользователь выберет адрес вне разрешенных зон, он увидит ошибку. Текст этой ошибки меняется здесь:
*   Файл: `apps/web/src/features/order-creation/api/ensure-city-action.ts`
*   Строка: `throw new Error("Ваш текст здесь");` (внутри блока `if (!isSfoRegion)`).

### 3. Как изменить привязку категорий услуг
По умолчанию все новые города получают доступ ко всем услугам. Если вы хотите это изменить:
*   Файл: `apps/web/src/features/order-creation/api/ensure-city-action.ts`
*   Найдите блок `db.cityCategory.createMany`.
*   Вы можете добавить фильтрацию в `db.category.findMany`, чтобы выбирать только нужные `id` категорий.
