# Архитектура географического охвата (Гео-модуль)

## Описание
Платформа использует гибридную модель данных, обеспечивающую масштабируемость на большие территории (ЮФО, СКФО и новые регионы) без необходимости ручного наполнения базы данных каждым населенным пунктом. Система автоматически регистрирует новые города при первом заказе.

---

## 1. Единый источник истины (Single Source of Truth)

С февраля 2026 года география проекта управляется централизованно. Больше не нужно вносить правки в разные файлы.

### Ключевые файлы конфигурации:
1.  **`apps/web/src/shared/config/geo.ts`** — **Главный узел управления**. Содержит правила валидации, список разрешенных регионов для DaData и тексты сообщений об ошибках.
2.  **`apps/web/prisma/seed-data.mjs`** — **Эталонный справочник**. Содержит список из 50+ опорных городов и категорий для первичного наполнения БД (Seed).

---

## 2. Зона покрытия (19+ субъектов РФ)

Сервер фильтрует все запросы и разрешает работу только в следующих регионах:
- **ЮФО**: Ростовская, Волгоградская, Астраханская обл., Краснодарский край, Адыгея, Калмыкия, Крым, Севастополь.
- **СКФО**: Ставропольский край, Дагестан, Чечня, Ингушетия, Северная Осетия, Кабардино-Балкария, Карачаево-Черкесия.
- **Новые регионы**: ДНР, ЛНР, Запорожская и Херсонская области.

## 3. Схема работы данных

```mermaid
graph TD
    User((Пользователь)) --> AddressField[Поле адреса в форме]
    AddressField --> API_Suggest[/api/suggest/address]
    API_Suggest --> GeoConfig[shared/config/geo.ts]
    GeoConfig -- Список регионов --> DaData{DaData API}
    DaData -- Подсказки --> User
    
    User -- Выбор адреса --> EnsureAction[ensureCityAction]
    EnsureAction --> GeoConfig_Val[isRegionSupported]
    GeoConfig_Val -- Валидация -- OK --> DB[(Database)]
    DB -- Создание города + привязка услуг --> Order[Создание заказа]
```

## 4. Как это работает (Техническая реализация)

### А. Серверный фильтр DaData
Весь поиск адресов проксируется через `/api/suggest/address`. Роут берет список `SUPPORTED_REGIONS` из конфига и отправляет его в DaData. Это гарантирует, что пользователь увидит подсказки **только** из разрешенных регионов.

### В. Ленивое обогащение данных (Lazy Enrichment)
Если город уже существует в базе (например, был добавлен через Seed без полных данных), но пользователь выбирает его через DaData:
- Система автоматически обновляет `fiasId`, `lat` и `lng` в существующей записи.
- Это позволяет базе данных самосовершенствоваться и повышать точность гео-поиска без ручного вмешательства.

## 5. Инструкция по расширению (Шпаргалка)

### Как добавить новый регион (например, Московская область)
Теперь это делается в **один шаг**:

1.  Откройте файл **`apps/web/src/shared/config/geo.ts`**.
2.  Добавьте название региона в массив `SUPPORTED_REGIONS`:
    ```typescript
    export const SUPPORTED_REGIONS = [
      // ... существующие
      "Московская" 
    ];
    ```
**Всё!** После этого:
- Поиск DaData начнет предлагать города Подмосковья.
- Сервер разрешит создание этих городов в базе.
- Сообщение об ошибке (если оно общее) останется актуальным.

### Как изменить текст сообщения об ограничении
Если вы хотите изменить фразу "На данный момент сервис работает только в...", просто поменяйте константу `GEO_LIMIT_MESSAGE` в том же файле `geo.ts`. Изменения применятся мгновенно во всем интерфейсе (формы создания, редактирования, мастер заказов).
