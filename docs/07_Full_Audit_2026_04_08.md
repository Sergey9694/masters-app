## Критические проблемы (обязательно к исправлению)

### 1. .env содержит реальные секреты в дереве проекта

**Файл:** .env — содержит реальные секреты:
- TELEGRAM_BOT_TOKEN — рабочий токен бота
- JWT_SECRET — секрет подписи сессий
- DADATA_API_KEY — API ключ DaData

Хотя .env в .gitignore, сам файл .env лежит в дереве проекта. Любой git clone на публичном репо утечёт секреты. **Токены уже скомпрометированы** (попадают в git log и могут быть в форках).

**Что делать:**
1. Ротировать все токены в .env прямо сейчас
2. Оставить только .env.example в репо
3. Добавить pre-commit hook для защиты от будущих утечек

---

### 2. Cron-эндпоинт без аутентификации

**Файл:** src/app/api/jobs/expire-tasks/route.ts (строки 7-11)

Код проверки закомментирован. Любой может вызвать /api/jobs/expire-tasks и архивировать заявки, создавая DoS.

**Что делать:** Раскомментировать и реализовать проверку CRON_SECRET.

---

### 3. Отсутствие Prisma binaryTargets для Alpine/Docker

**Файл:** prisma/schema.prisma (строка 3)

В Dockerfile используется node:20-alpine (musl libc), но binaryTargets = ["native"] генерирует бинарник для архитектуры билда. Это может вызвать рантайм-ошибку libquery_engine.so.node: not found.

**Что делать:** Добавить binaryTargets = ["native", "debian-openssl-3.0.x"].

## Серьёзные проблемы (рекомендуется исправить)

### 4. Гонка при подключении к БД в dev-режиме

**Файл:** src/shared/lib/db.ts

При HMR (Hot Module Replacement) в dev-режиме может создаваться несколько экземпляров PrismaClient без proper cleanup. Хотя есть паттерн globalForPrisma, отсутствует отключение пула при отключении.

**Что делать:** Добавить db.$disconnect() в cleanup.

---

### 5. address поле в taskSchema — обязательное с минимумом 5 символов

**Файл:** src/features/task-creation/model/task-schema.ts (строка 14)

Задача не может быть создана без адреса. Это ограничивает удалённые задачи (консультации, IT-поддержка).

**Что делать:** Сделать .optional() или .nullable().

---

### 6. Rate Limit — in-memory, не работает в multi-instance

**Файл:** src/shared/lib/rate-limit.ts

In-memory Map не работает в multi-instance деплое и сбрасывается при рестарте. Для standalone режима Docker — один инстанс, но при масштабировании это станет проблемой.

**Что делать:** Заменить на Redis или хотя бы добавить предупреждение в README.

---

### 7. Порт БД в .env не совпадает с .env.example

**Файл:** .env — порт 5435
**Файл:** .env.example — порт 5434

Это несоответствие между примером и реальной конфигурацией.

---

### 8. X-Frame-Options и CSP конфликтуют

**Файл:** next.config.ts (строки 19, 39)

X-Frame-Options: ALLOW-FROM https://web.telegram.org — этот заголовок не поддерживается современными браузерами (только CSP frame-ancestors работает). При этом CSP уже содержит frame-ancestors self https://web.telegram.org.

**Что делать:** Убрать X-Frame-Options или заменить на DENY (CSP уже покрывает).

---

### 9. startup.js зависит от next build

**Файл:** scripts/startup.js (строка 56)

Путь ./server.js зависит от next.config.ts -> output: standalone. Файл .next/standalone/server.js существует только после next build. В dev-режиме этот скрипт не работает.

---

### 10. LocationFilter использует внешний IP-API без ключа

**Файл:** src/features/geo-search/ui/LocationFilter.tsx (строка 62)

Это сторонний API без API ключа и без ограничения rate. Также может быть заблокирован CORS.

**Что делать:** Добавить fallback или использовать собственный эндпоинт.

## Minor issues

### 11. prisma/seed.mjs и prisma/seed.ts — два seed файла

package.json указывает seed: tsx prisma/seed.ts, но startup.js вызывает node prisma/seed.mjs. Если оба файла существуют и различаются, будет непредсказуемое поведение.

### 12. taskSchema использует address как строку, но Prisma имеет taskLocation: geometry

Поле taskLocation в Prisma (Unsupported("geometry(Point, 4326)")) никогда не заполняется при создании задачи — адрес хранится только как строка. PostGIS-запрос getTasksNearby никогда не вернёт данные, т.к. taskLocation всегда null.

### 13. createSafeAction — хендлер бросает ошибку вместо возврата error

**Файл:** src/shared/lib/create-safe-action.ts (строка 27)

Хендлер бросает Error, который ловится и возвращается как error. Но createOrderAction использует этот паттерн и возвращает redirect внутри data — это не типизировано в ActionState. Тип TOutput не имеет redirect.

### 14. Отсутствует CRON_SECRET в .env.example

### 15. useHaptics не проверяет наличие Telegram WebApp

**Файл:** src/shared/lib/telegram/use-haptics.ts — если вызывается вне Telegram, может вызвать ошибку.

### 16. help.txt и prisma_error.txt в корне репозитория — временные файлы

Не должны быть в репозитории. Добавить в .gitignore.
