# Концепт: Бизнес-витрина (BusinessProfile)

> Дата: 2026-04-26
> Статус: Концепт, ожидает спецификации

---

## Суть идеи

Отдельный режим для малого бизнеса на платформе: кондитеры, детейлинг, фотостудии, мастерские — все кто имеет **бренд**, но слишком маленький для отдельного сайта.

Закрывает сегмент который нигде нормально не представлен: это не фрилансер-одиночка, но и не крупная компания.

---

## Ключевое решение

**Бизнес-профиль — апгрейд Provider-аккаунта, не отдельный тип.**

Один и тот же человек может:
- Откликаться на заказы в общей ленте как исполнитель
- Иметь витрину своего бизнеса в отдельном разделе
- Публиковать свои услуги (ServiceListing) в общем каталоге

Нет раздвоения аккаунтов. Один аккаунт — два режима отображения.

---

## Структура витрины

Фиксированные секции, визуально богатые. Не конструктор блоков (MVP).

Принцип: хозяин должен захотеть скинуть ссылку на свою страницу в Telegram или Instagram. Страница выглядит как настоящий сайт, не как карточка в справочнике.

```
┌─────────────────────────────────────┐
│  Обложка (cover photo)              │
│  Лого + Название бизнеса            │
│  Тег-категория · Город · ★ Рейтинг  │
├─────────────────────────────────────┤
│  О нас (до 500 символов)            │
├─────────────────────────────────────┤
│  Портфолио (галерея фото/видео)     │
│  — главный блок, самый большой      │
├─────────────────────────────────────┤
│  Услуги и цены (список с ценами)    │
├─────────────────────────────────────┤
│  Отзывы (только по реальным заказам)│
├─────────────────────────────────────┤
│  Режим работы · Соцсети · Контакт   │
└─────────────────────────────────────┘
```

---

## Поля BusinessProfile (модель данных)

| Поле | Тип | Описание |
|---|---|---|
| `businessName` | string | Название бизнеса |
| `logo` | string (url) | Логотип |
| `coverImage` | string (url) | Обложка страницы |
| `tagline` | string (150 симв.) | Короткий слоган |
| `description` | string (500 симв.) | О нас |
| `portfolio` | Media[] | Галерея фото/видео |
| `services` | ServiceItem[] | Список услуг с ценами |
| `workingHours` | JSON | Расписание по дням |
| `socialLinks` | JSON | TG, VK, Instagram, сайт |
| `categoryId` | FK → Category | Основная категория |
| `cityId` | FK → City | Город |
| `serviceArea` | string | Зона обслуживания |
| `isVerified` | boolean | Верификация |
| `slug` | string (unique) | URL витрины |

Связь: `ProviderProfile` 1:1 `BusinessProfile` (опциональная).

---

## Где бизнесы появляются на платформе

1. **Отдельный раздел** `/businesses` — каталог витрин с фильтрами по категории и городу
2. **Карточки в общих списках** — при поиске исполнителей бизнес-аккаунты отмечаются бейджем "Бизнес"
3. **Их ServiceListing'и** — попадают в общий каталог `/services` как обычно
4. **Прямая ссылка** `/b/[slug]` — для расшаривания

---

## Вирусный механизм

Каждая витрина — это персональная страница которую бизнес расшаривает сам:
в Telegram, Instagram, визитках, клиентам. Это бесплатный трафик на платформу.

---

## Монетизация

| Инструмент | Механика |
|---|---|
| Базовая витрина | Бесплатно (привлечение бизнесов) |
| Продвижение в каталоге | Топ в `/businesses` — платно |
| Премиум-функции | Аналитика просмотров, кастомный slug, приоритет в поиске |

---

## Фазы реализации

**MVP (Фаза 1):**
- Форма создания бизнес-профиля (апгрейд в настройках)
- Страница витрины `/b/[slug]`
- Каталог `/businesses` с базовыми фильтрами
- Бейдж "Бизнес" в карточках исполнителей

**Фаза 2:**
- Конструктор блоков (перестановка секций)
- Аналитика просмотров витрины
- Абонементы / пакеты услуг на витрине

---

## Конкурентное преимущество

Ни Profi.ru, ни YouDo, ни Avito не дают малому бизнесу полноценную витрину.
Avito есть "магазин" — но он для товаров, не для услуг.
Это незанятая ниша на российском рынке услуг.
