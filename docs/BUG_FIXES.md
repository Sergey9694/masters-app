# Реестр исправленных багов (Bug Fix Log)

В этом файле фиксируются критические ошибки, их причины и принятые решения для предотвращения рецидивов.

## [2026-04-22] Исправление регистрации мастера и ленты заказов

### 1. Дублирование окна выбора файлов (Double File Dialog)
**Описание:** При нажатии на зону загрузки портфолио в форме регистрации мастера окно выбора файлов открывалось дважды.
**Причина:** 
- Компонент `Field` был реализован через тег `<label>`, внутри которого находился `div` с обработчиком `onClick`, вызывающим `input.click()`. 
- В HTML клик по `label` автоматически триггерит связанный `input`. В итоге срабатывал и JS-обработчик, и нативное поведение браузера.
**Решение:**
- Корневой элемент `Field` изменен с `label` на `div` (файл `ProviderRegistrationFormLight.tsx`).
- Добавлено состояние `isProcessingFiles` для блокировки повторных кликов во время обработки (HEIC -> JPEG).

### 2. Пустая лента заказов для новых мастеров (Smart Feed Lock)
**Описание:** Мастера после регистрации видели пустую ленту, если в их категориях еще не было заказов. Кнопка «Все категории» не помогала, так как сбрасывала фильтр в URL, что снова включало «Умную ленту» по категориям профиля.
**Причина:** 
- В `order.service.ts` логика была жестко завязана: если `categoryId` отсутствует — фильтруем по профилю мастера. 
- В интерфейсе отсутствовал способ явно запросить «всё без исключений».
**Решение:**
- Введен зарезервированный параметр `all` для `categoryId`.
- Обновлен `OrdersFilters`: добавлен пункт «✨ Моя лента» (сброс в умный дефолт) и «Все категории» (параметр `all`).
- В `OrderFeedClient` добавлены «умные заглушки»: если заказов нет, пользователю предлагается кнопка «Посмотреть все активные заказы», которая принудительно отключает фильтры.

### 3. Медленная обратная связь при загрузке фото
**Описание:** При выборе тяжелых фото (iPhone HEIC) казалось, что ничего не происходит, пока шла конвертация на клиенте.
**Решение:** 
- Внедрен индикатор загрузки с текстом «Обработка изображений...» в реальном времени.

## [2026-04-22] Внедрение SEO-ссылок и порядковых номеров заказов

### 1. Переход на SEO-дружелюбные URL (Slugs)
**Описание:** Ранее ссылки на заказы содержали технические ID (CUID), например `/orders/cmoa...`. Это плохо для SEO и неудобно для пользователей.
**Решение:**
- В модель `Order` добавлены поля `orderNumber` (автоинкремент) и `slug` (уникальная строка).
- Создана утилита `slugify.ts` для транслитерации кириллицы.
- При создании заказа генерируется слаг формата `nazvanie-zakaza-123`.
- Роутинг изменен с `orders/[id]` на `orders/[slug]`.
- Реализована обратная совместимость: поиск заказа в БД теперь идет по ID или по слагу (через `OR`).

### 2. Порядковые номера заказов (Human-friendly IDs)
**Описание:** У заказов не было короткого идентификатора для общения с поддержкой или вывода в статистику.
**Решение:**
- Использован тип `Int @unique @default(autoincrement())` в Postgres.
- Номера заказов выведены во всех ключевых местах: в ленте, в списке «Мои заказы» и на детальной странице заказа в формате `№123`.
