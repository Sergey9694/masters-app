# План реализации: Карта в стиле Avito (Dynamic BBox + Custom Layouts)

## 1. Цель
Трансформировать текущую реализацию карты в высокопроизводительный и визуально привлекательный инструмент поиска заказов, соответствующий стандартам индустрии (Avito, Airbnb).

## 2. Архитектурные изменения

### Backend: Bounding Box API
- **Задача**: Расширить метод `orderService.listMapPoints` для фильтрации по координатам прямоугольника (BBox).
- **Технология**: PostGIS `ST_MakeEnvelope`.
- **Параметры**: `latMin, lngMin, latMax, lngMax`.
- **Безопасность**: Валидация размера области через Zod. Ограничение на максимальную площадь (нельзя запросить всю страну сразу).

### Frontend: Dynamic Loading
- **Собыствие**: Слушать `boundschange` у карты.
- **Оптимизация**: Debounce запросов (300ms) для предотвращения спама при перемещении.
- **Кэширование**: Использование `React Query` (или простого локального кэша) для предотвращения повторной загрузки одних и тех же областей.

## 3. UI/UX: Custom Layouts

### Кастомные метки (Placemarks)
- **Вид**: Прямоугольная плашка с закругленными углами.
- **Контент**: Бюджет заказа (например, "5 000 ₽").
- **Состояние**: Активный цвет (фиолетовый) при выборе, серый/белый в обычном состоянии.
- **Технология**: `ymaps.templateLayoutFactory.createClass`.

### Кастомные кластеры
- **Вид**: Укрупненная плашка.
- **Контент**: Количество заказов и минимальная цена ("12 заказов от 1 000 ₽").
- **Поведение**: Зум при клике или открытие списка (зависит от уровня зума).

## 4. Deep Linking & State Sync
- Синхронизация `center`, `zoom` и `bbox` с URL.
- Пример: `/orders/map?lat=47.22&lng=39.71&z=12`.
- Base64-кодирование параметров (опционально, для краткости как у Avito).
