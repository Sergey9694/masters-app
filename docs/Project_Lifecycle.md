# Жизненный цикл проекта (Project Lifecycle)

## Фаза 1: Идея и Планирование (Начало Марта 2026)
- **Цель**: Решить проблему поиска доверенных мастеров в микрорайоне.
- **Действие**: Написание `implementation_plan.md`. Согласование стека (Next.js 16 + Tailwind 4 + PostGIS).
- **Архитектура**: Выбор Feature-Sliced Design (FSD) как стандарта для поддержания Enterprise-уровня.

## Фаза 2: Инфраструктурный взлет (Середина Марта 2026)
- **Развертка**: Настройка Docker-контейнеров для локальной разработки.
- **База данных**: Первая миграция Prisma, подключение PostgreSQL + PostGIS. Столкнулись с первыми трудностями PostGIS (типы `Unsupported`).
- **Nginx & Proxy**: Настройка прокси-сервера для локальной разработки, чтобы эмулировать продакшн-окружение и корректно работать с HTTPS (важно для Telegram TWA).

## Фаза 3: Ядро системы (Конец Марта 2026)
- **Авторизация**: Реализация сложного флоу авторизации через Telegram (валидация подписи, сессии на httpOnly cookies).
- **Entities**: Создание фундаментальных сущностей `User` и `MasterProfile`. Верификация связи 1:1.
- **Features**: Первая версия функционала создания задачи (`task-creation`). Реализована премиальная форма ввода данных.

## Фаза 4: Расширение UI и Гео-задачи (Текущий этап, Апрель 2026)
- **Widgets**: Создание `CategoryGrid` и начальная версия `TaskFeed`.
- **Styling**: Полный переход на Tailwind 4 (использование CSS-переменных для темизации).
- **Issues**: Обнаружена несовместимость Prisma 7 (удален `url` из схемы), начат процесс адаптации.

## Будущие фазы разработки (План на Май 2026)
- **Уведомления**: Подключение Telegram Bot для мгновенной рассылки заказов мастерам в реальном времени.
- **Масштабирование**: Тестирование поиска по гео-координатам на больших объемах данных (свыше 10k заказов в день).
- **Релиз**: Финализация MVP и первый запуск в тестовом микрорайоне.
