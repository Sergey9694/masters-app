## 2. Детальный разбор изменений

### 2.1. Административный "Security Gate"
Все критические операции (удаление пользователей, смена ролей, верификация исполнителей) переведены на использование `adminActionClient`.
- **Гарантия:** Проверка роли `ADMIN` выполняется на стороне сервера перед любым доступом к данным.
- **Логирование:** Каждое действие фиксируется в таблице `AuditLog`.

### 2.2. Распределенный Rate Limiting
Система переведена с локального `Map` (который сбрасывался при перезапуске Docker-контейнера) на **Redis**.
- **Стабильность:** Ограничения сохраняются между деплоями и работают согласованно в кластере.
- **Производительность:** Асинхронные операции с Redis через `ioredis`.

### 2.3. Унифицированный Audit Log
Новый сервис `src/shared/lib/audit.ts` автоматически:
- Извлекает IP-адрес клиента из `x-forwarded-for`.
- Сохраняет структурированные метаданные (например, ID удаленного пользователя или старая/новая роль).

### 2.4. Типизация (Zero Any Policy)
- Устранены все `any` в интеграции с Dadata (геолокация).
- Исправлены `any` в основных компонентах просмотра заказов.
- Использованы типы Prisma (`OrderWithDetails`) для гарантии целостности данных.
