# Известные проблемы и ошибки (Issue Tracker)

## Текущие вызовы (Technical Challenges)

### 1. Ошибка Prisma 7 (P1012): `url` в схеме больше не поддерживается
**Описание**: В Prisma 7 произошли радикальные изменения в работе с источниками данных. Прямое указание `url = env("DATABASE_URL")` в `schema.prisma` выдает ошибку валидации.
**Статус**: 🛠 В процессе исправления.
**Решение**: Необходимо перенести конфигурацию в `prisma.config.ts` и использовать Driver Adapter. Все `PRISMA_CLIENT` теперь инициализируются через адаптер `pg`.

### 2. Поддержка PostGIS в Prisma
**Описание**: Prisma до сих пор не поддерживает типы геометрии (Point) «из коробки». Они определены как `Unsupported("geometry(Point, 4326)")`.
**Риск**: Прямое обновление этих полей через Prisma Client (например, `create`) может не работать стандартным образом.
**Решение**: Использовать `prisma.$queryRaw` для обновлений и расширенного поиска. В будущем возможен переход на другой ORM (например, Drizzle), если Prisma 7 не улучшит поддержку геометрии.

### 3. Загрузка фото (Photo Upload)
**Описание**: В данный момент функционал загрузки изображений для заданий отсутствует.
**Решение**: Планируется использование `sharp` для оптимизации (конвертация в WebP) и временное хранение на локальном диске Docker-контейнера перед переходом на S3/MinIO.

### 4. Синхронизация данных пользователя
**Описание**: При первом входе через Telegram необходимо гарантировать создание записи в БД. В условиях высокого параллелизма возможны Race Conditions при проверке существования пользователя.
**Решение**: Использовать атомарные `upsert` операции в Prisma или транзакции на уровне БД.

### 5. Docker: Windows Path Issues
**Описание**: При разработке на Windows возможны проблемы с маппингом томов (volumes) в Docker.
**Решение**: Рекомендуется использовать WSL2 или запускать скрипты через `dev.ps1` (настроен в `package.json`).
