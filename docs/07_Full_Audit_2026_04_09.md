## Критические проблемы (P0 — обязательно к исправлению)

### 1. Cron-эндпоинт без аутентификации

**Файл:** `src/app/api/jobs/expire-tasks/route.ts` (строки 7-11)

Проверка `CRON_SECRET` закомментирована. Любой, кто знает URL, может вызывать endpoint и архивировать задачи (DoS).

**Что делать:** Раскомментировать проверку `CRON_SECRET`.

---

### 2. Файлы с секретами в git — help.txt и prisma_error.txt

**Файлы:** `help.txt`, `prisma_error.txt` в корне репозитория

`prisma_error.txt` содержит строку подключения к БД с логином и паролем (`postgresql://admin:admin_pass@localhost:5434/masters_db`). Оба файла tracked в git, нет в `.gitignore`.

**Что делать:**
1. `git rm --cached help.txt prisma_error.txt`
2. Добавить `*.txt` в `.gitignore`
3. Закоммитить

## Серьёзные проблемы (P1 — рекомендуется исправить)

### 3. Отсутствие binaryTargets для Alpine/Docker

**Файл:** `prisma/schema.prisma` (строка 3)

`binaryTargets = ["native"]` — при билде на Windows/macOS Alpine-бинарник не будет сгенерирован. Dockerfile использует `node:20-alpine`.

**Что делать:** Добавить `binaryTargets = ["native", "linux-musl-openssl-3.0.x"]`.

---

### 4. taskLocation (PostGIS) никогда не заполняется

**Файлы:**
- `prisma/schema.prisma:75` — `taskLocation Unsupported("geometry(Point, 4326)")?`
- `src/features/task-creation/api/create-task-action.ts` — не пишет geometry
- `src/entities/task/api/task-geo.ts` — `getTasksNearby` использует `ST_DWithin`

Geo-поиск всегда возвращает 0 результатов, т.к. `taskLocation` всегда `NULL`.

**Что делать:** Добавить lat/lng в форму создания задачи и записать через `$executeRaw` + `ST_SetSRID(ST_MakePoint(...))`.

---

### 5. address обязателен в taskSchema

**Файл:** `src/features/task-creation/model/task-schema.ts` (строка 14)

Нельзя создать удалённую задачу (консультация, IT-поддержка).

**Что делать:** Сделать `.optional()`.

---

### 6. X-Frame-Options: ALLOW-FROM не поддерживается

**Файл:** `next.config.ts` (строка 20)

`ALLOW-FROM` игнорируется всеми современными браузерами. CSP `frame-ancestors` (строка 40) уже покрывает этот кейс.

**Что делать:** Заменить на `DENY` или удалить.

---

### 7. Порт БД в .env.example не совпадает с .env

**Файлы:** `.env` (порт 5435) vs `.env.example` (порт 5434)

**Что делать:** Синхронизировать `.env.example` с реальным Docker Compose.

---

### 8. uploadImagesAction глотает ошибки

**Файл:** `src/features/task-creation/api/upload-action.ts` (строки 53-56, 61-63)

`catch { console.error(err) }` — ошибка логируется, но пользователь получает успешный ответ с неполным списком изображений.

**Что делать:** Возвращать `error` в результате при сбойе upload.
