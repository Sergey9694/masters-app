## Отложено / Не требует исправления

### PostGIS taskLocation — отложено

**Статус:** ⏸️ Отложено (PostGIS пока не нужен)

Поле `taskLocation` (geometry) не заполняется при создании задачи, geo-поиск не работает.
Когда PostGIS понадобится: добавить lat/lng в форму создания задачи и записать через `$executeRaw` + `ST_SetSRID(ST_MakePoint(...))`.

### Два seed-файла — осознанное разделение

**Статус:** ✅ Не требует исправления

- `prisma/seed.ts` — для dev (`tsx prisma/seed.ts` через package.json)
- `prisma/seed.mjs` — для production-контейнера (`node prisma/seed.mjs` через startup.js)

Разделение обосновано: в production-контейнере нет `tsx`, только Node.js.

### Inline .filter() в JSX — низкий приоритет

**Статус:** 🟢 Низкий приоритет

`my-responses/page.tsx` и `my-tasks/page.tsx` используют inline `.filter()` в JSX.
При текущих объёмах данных (< 50 задач) это не влияет на производительность.
Оптимизировать через `useMemo` при росте данных.

### masterProfile! non-null assertions — допустимо

**Статус:** 🟢 Допустимо

`my-responses/page.tsx` использует `user.masterProfile!.id` в ветках, где `masterProfile` гарантированно существует.
Non-null assertion здесь оправдан, но можно заменить на локальную переменную при рефакторинге.
