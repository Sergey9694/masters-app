# 🎭 Workflow: Task Delegation (Orchestrator)

Этот воркфлоу активирует роль Оркестратора для декомпозиции и делегирования сложных задач.

## Шаги

1. **Инициализация**:
   - Прочитать `.agent/skills/agent-protocol/SKILL.md`.
   - Прочитать `.agent/skills/agent-orchestrator/SKILL.md`.

2. **Анализ задачи**:
   - Оценить масштаб задачи и затронутые модули.
   - Свериться с `docs/PROJECT_STATE.md`.

3. **Декомпозиция**:
   - Разбить задачу на подзадачи (Research, Arch, Security, Code, Test).
   - Выбрать подходящих агентов для каждой подзадачи.

4. **Представление плана**:
   - Показать план пользователю и дождаться одобрения.

5. **Выполнение**:
   - Последовательно выполнять подзадачи, переключая роли (скиллы).

6. **Ревью и Контроль Качества** (Role: Code Quality + QA + Security):
   - Активировать `agent-code-quality` для проверки кода и типов.
   - Активировать `agent-qa` для верификации сценариев.
   - Активировать `agent-security` для финального подтверждения безопасности.

7. **Отчетность и Синхронизация** (Role: Notary):
   - Активировать `agent-notary`.
   - Сгенерировать детальный отчет в `docs/agent-reports/`.
   - Обновить `PROJECT_STATE.md`, `DEVELOPMENT_PLAN.md` и `docs/BUG_FIXES.md`.

8. **Финальная Сдача** (Role: Orchestrator):
   - Предоставить владельцу понятное резюме проделанной работы.
   - Дать инструкции по проверке результата.
